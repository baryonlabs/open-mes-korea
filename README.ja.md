# Open MES Korea

**小さく、堅牢で、拡張可能なオープンソースMES**

韓国の製造現場における実際の業務フローを出発点とし、
国・言語・業種を越えて拡張できる製造実行システムを目指します。

[한국어](README.md) · [English](README.en.md) · [简体中文](README.zh-CN.md)

> ではなく、開発中にAPIや文書が変更される可能性があります。

## 目標

Open MES KoreaはERP全体を置き換えるのではなく、次の領域に集中します。

```text
製造現場の実行
  + LOTトレーサビリティ
  + 製造データの収集・分析
  + 統制されたAI
```

韓国の製造用語、作業方法、導入条件を現実的な出発点とします。ただし、
UI、文書、業務用語、現場固有のルールは多言語・多地域へ拡張できる
構造にします。基本言語は韓国語です。

## データ収集のゴールデンタイム

製造AIの第一歩はモデルやベンダーの選定ではなく、今発生している現場
データを失う前に収集することです。作業者入力、バーコード、CSV、標準
APIから始め、PostgreSQL、CSV、APIで移植可能な履歴を保持します。
これにより、長い製品選定後に収集をやり直さず、自社データでAIモジュール
とベンダーを比較できます。

## 設計原則

- **小さなコア:** 共通する製造実行フローだけを内蔵
- **多言語:** 韓国語を基本とし、追加言語を拡張可能にする
- **拡張性:** ERP、設備、分析、業種固有ルールを独立して接続
- **安定性:** 監査ログ、重複防止、訂正履歴、障害復旧
- **堅牢性:** 生産データの整合性と明示的なLOT genealogyを保持
- **高速性:** 現場入力と運用照会の応答性を重視
- **安全なAI:** 照会、提案、承認、実行の権限を分離

## 基本フロー

```text
品目 / BOM / 工程
→ 作業指示
→ 作業開始・終了
→ 生産数・不良記録
→ 原材料LOT投入
→ 製品LOT生成
→ 生産状況・LOT追跡
→ 基本分析・AI照会
```

## コア

- 品目、BOM、工程、ルーティング、作業者、設備
- 作業指示、工程実績、生産数、不良
- 原材料投入、製品LOT、LOT genealogy
- 手入力、バーコード、CSV、認証済みHTTP ingestion
- 生産量、不良率、工程時間の基本分析
- ユーザー、権限、監査ログ、event outbox

## 拡張機能

- ERP、WMS、QMSとの連携
- MQTT、OPC UA、Modbus、Serial、PLC connector
- Phoenix Broadwayによる高スループット収集
- TimescaleDBまたはClickHouseによるtelemetry分析
- OEE、高度計画、予測分析、業種別ダッシュボード
- AI照会、提案、承認ベースのアクション

Broadwayは分析エンジンではなく、並行処理、バックプレッシャー、
バッチ、再試行、障害分離を担当します。

## 製造AI拡張モジュール

- 外観、寸法、組立、X-ray/CTを対象としたビジョン検査
- 仮想計測、デジタルツイン、仮想試運転
- 工程変数、歩留まり、エネルギーの最適化
- AMR/AGV経路最適化とビジョンピッキング
- 需要・在庫予測とAPSスケジューリング
- 予知保全と設備異常検知
- 保護具、危険区域、危険行動の検知
- マニュアル、作業標準、トラブル対応用の製造LLM

これらはコア機能ではなく選択可能な拡張です。設備制御、品質判定、
作業者安全に関する判断には、現場検証、fail-safe、人の承認、監査記録を
必須とします。
