# 도입 의사결정 기준

이 문서는 Open MES Korea 도입 여부를 결정하는 기준을 제공한다.

## 결정 유형

| 결정 | 의미 |
|---|---|
| 도입 | MVP 범위로 실제 운영을 시작한다 |
| PoC | 한 라인/한 공정/한 제품군에서 검증한다 |
| 부분 도입 | LOT 추적, 작업지시, 현장 입력 중 일부만 시작한다 |
| 보류 | 기준정보/업무 표준화가 먼저 필요하다 |
| 부적합 | 현재 문제는 MES보다 ERP/회계/인사/설비제어 쪽이다 |

## 도입 권장 조건

아래 조건을 많이 만족할수록 도입 가치가 높다.

- 작업지시 진행이 보이지 않는다.
- 현장 실적이 늦게 취합된다.
- LOT 추적이 어렵다.
- 불량 원인 분석이 어렵다.
- ERP가 현장 실행을 충분히 담지 못한다.
- 작업자가 바코드/태블릿 입력을 할 수 있다.
- 품목/BOM/공정 기준정보가 어느 정도 있다.
- 경영진이 현장 데이터 기반 의사결정을 원한다.
- AI 도입 전에 제조 데이터를 정리하려 한다.

## 보류 권장 조건

아래 조건이 많으면 바로 도입보다 준비가 먼저다.

- 품목 코드가 없다.
- BOM과 공정 순서가 문서화되어 있지 않다.
- 현장 작업 방식이 매일 크게 바뀐다.
- 작업자가 시스템 입력을 할 수 없는 환경이다.
- 네트워크/장비 설치가 전혀 불가능하다.
- 기존 ERP/MES와 역할 분담이 정해지지 않았다.
- 도입 책임자와 현장 champion이 없다.

## PoC 범위 추천

처음 PoC는 작아야 한다.

권장:

- 한 제품군
- 한 생산라인
- 2~4개 공정
- 1개월 이내
- 작업지시 20~100건
- LOT 추적이 필요한 대표 흐름 1개

PoC 성공 기준:

- 작업자가 작업 시작/종료를 입력한다.
- 생산관리자가 당일 진행 현황을 본다.
- 자재 LOT와 제품 LOT가 연결된다.
- 불량 코드와 수량이 기록된다.
- 엑셀 작업일보 일부가 사라진다.
- 클레임 상황을 가정해 LOT 역추적이 가능하다.

## 의사결정 표

| 항목 | 점수 0 | 점수 1 | 점수 2 |
|---|---|---|---|
| 작업지시 관리 | 없음 | 엑셀/종이 | 시스템화 가능 |
| 현장 입력 | 불가능 | 일부 가능 | 태블릿/스캔 가능 |
| LOT 필요성 | 낮음 | 일부 | 높음 |
| 기준정보 | 없음 | 일부 | 정리됨 |
| 불량 관리 | 사후 집계 | 일부 기록 | 공정별 기록 가능 |
| ERP 연동 | 불필요 | 향후 필요 | 현재 필요 |
| 설비/IoT | 없음 | 일부 | 연동 가능 |
| AI 활용 의지 | 없음 | 관심 | 명확한 use case 있음 |

판정:

| 총점 | 권장 |
|---:|---|
| 0~5 | 보류 |
| 6~9 | PoC 전 기준정보 정리 |
| 10~13 | PoC 권장 |
| 14~16 | 단계적 도입 권장 |
