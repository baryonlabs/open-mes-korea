# 도입 리스크와 대응

MES 도입 실패는 대부분 소프트웨어 기능 부족보다 업무 범위, 데이터, 현장 입력, 책임 경계 문제에서 발생한다.

## 주요 리스크

| 리스크 | 증상 | 영향 | 대응 |
|---|---|---|---|
| 기준정보 부족 | 품목/BOM/공정이 계속 바뀜 | 작업지시 생성 불가 | MVP 전 기준정보 정리 |
| 현장 입력 저항 | 작업자가 입력을 회피함 | 데이터 신뢰도 하락 | 태블릿 UX, 바코드, 최소 입력 |
| 과도한 범위 | ERP/재고/회계/설비제어까지 한 번에 요구 | 일정 지연 | 현장 실행 + LOT 추적으로 제한 |
| LOT 규칙 불명확 | LOT 번호가 중복되거나 누락 | 추적성 실패 | LOT 발행 규칙 먼저 정의 |
| 불량 코드 과다 | 작업자가 불량 유형 선택을 못함 | 품질 데이터 품질 저하 | 1차 코드는 10~30개로 시작 |
| ERP 책임 경계 불명확 | 어느 시스템이 기준인지 충돌 | 데이터 불일치 | system of record 정의 |
| 설비 연동 과신 | PLC 데이터만 있으면 실적이 된다고 봄 | 잘못된 자동 집계 | candidate로 저장 후 매칭 |
| 네트워크 불안정 | 현장 입력 누락 | 운영 신뢰 저하 | 오프라인 큐/edge buffer |
| AI 오판 | 잘못된 제안 또는 근거 부족 | 신뢰 하락 | read-only, 근거 링크, 승인 기반 |
| 보안 미흡 | 설비망/DB/API 노출 | 심각한 사고 | device token, mTLS, outbound 구조 |

## 실패를 줄이는 도입 순서

1. 현장 업무 조사
2. 품목/BOM/공정/LOT 기준 정리
3. 한 라인 PoC
4. 작업자 UX 검증
5. LOT genealogy 검증
6. ERP/IoT 연동
7. AI context API
8. AI approval workflow

## 하지 말아야 할 도입 방식

- 첫날부터 모든 공정과 모든 품목을 넣는다.
- 작업자 교육 없이 시스템만 설치한다.
- 종이/엑셀 양식을 그대로 화면으로 옮긴다.
- 설비 데이터를 생산실적으로 자동 확정한다.
- AI가 승인 없이 작업지시나 LOT 데이터를 바꾸게 한다.
- ERP와 MES 중 어느 쪽이 기준인지 정하지 않는다.

## 성공 패턴

- 작은 범위에서 시작한다.
- 작업자 입력을 3초 안에 끝내도록 만든다.
- LOT 추적이 필요한 대표 제품군을 먼저 고른다.
- 불량 코드는 적게 시작하고 운영하면서 늘린다.
- 경영진 보고서보다 현장 데이터 입력 성공률을 먼저 본다.
- AI는 조회와 요약부터 시작한다.
