# 데이터 준비 체크리스트

MES 도입 성공은 소프트웨어보다 기준정보 품질에 크게 좌우된다.

## 필수 데이터

| 데이터 | 준비 여부 | 최소 필요 항목 | 비고 |
|---|---:|---|---|
| 품목 |  | 품목코드, 품명, 유형, 단위 | 원자재/반제품/완제품 구분 |
| BOM |  | 상위품목, 하위품목, 소요량, 단위 | 손실률은 선택 |
| 공정 |  | 공정코드, 공정명, 순서 | 회사 용어 유지 가능 |
| 라우팅 |  | 품목, 공정, 순서 | 작업지시 공정 생성에 필요 |
| 작업자 |  | 이름, 사번/ID, 역할 | 권한과 감사 로그에 필요 |
| 설비 |  | 설비코드, 설비명, 라인/공정 | 초기에는 선택 |
| 불량 코드 |  | 코드, 명칭, 적용 공정 | 품질 분석에 필요 |
| LOT 규칙 |  | LOT 번호 형식, 발행 시점 | 추적성 핵심 |

## 데이터 품질 점검

| 질문 | 예 | 아니오 | 조치 |
|---|---:|---:|---|
| 품목 코드가 중복 없이 관리된다 |  |  | 코드 정비 필요 |
| 같은 품목을 부서마다 다른 이름으로 부르지 않는다 |  |  | 표준 명칭 필요 |
| BOM 최신본이 명확하다 |  |  | 승인된 BOM 기준 필요 |
| 공정 순서가 품목별로 정리되어 있다 |  |  | 라우팅 정리 필요 |
| 불량 코드가 너무 세분화되어 있지 않다 |  |  | 현장 입력 가능한 수준으로 정리 |
| LOT 번호 생성 규칙이 일관적이다 |  |  | 자동 생성 규칙 정의 |
| 엑셀 원본의 책임자가 정해져 있다 |  |  | 데이터 owner 지정 |

## 데이터 준비 수준

| 수준 | 상태 | 권장 조치 |
|---|---|---|
| Level 0 | 품목/공정 기준이 없음 | 업무 표준화부터 진행 |
| Level 1 | 엑셀로 품목/공정 관리 | import 템플릿으로 PoC 가능 |
| Level 2 | BOM/라우팅 일부 정리 | 작업지시 MVP 가능 |
| Level 3 | LOT/불량 코드 정리 | 추적성 MVP 가능 |
| Level 4 | ERP 기준정보 존재 | ERP 연동형 도입 검토 |
